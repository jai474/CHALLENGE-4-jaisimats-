How this works

This is a fully standalone, offline build — plain HTML/CSS/JS, no server, no install, no build step (the logic lives in a small companion logic.js file so it can be unit-tested — see tests.html). The Concierge answers in your selected language and grounds every answer in this page's live simulated venue data (gate crowding, shuttle ETAs, parking, sustainability), so it never invents a number. It runs on a built-in rule-based GenAI-simulation engine — the same logic pattern a real deployment would wire to a live Claude model on a backend.

PromptWars · Challenge 4 — Smart Stadiums & Tournament Operations

What makes this build different

Most GenAI-assisted submissions to this brief converge on the same shape: color-coded gate bars, a chatbot, a wayfinding form. This build keeps all of that, but goes further in four concrete, checkable ways — every claim below is something you can watch happen live in the app, not just read about.

1. Real tournament data, not placeholders

The venue selector at the top of both apps switches between all 16 official FIFA World Cup 2026 host stadiums — real neutral tournament names, real host cities, real FIFA-confirmed capacities (43,036 to 80,824+), sourced from FIFA's official capacity announcement. Changing venues live-rescales every gate, the capacity gauge, and every density calculation in the app. Nothing here is a fictional "Stadium A."

2. A visible multi-agent simulation, not a random number

Turnstile entry counts aren't Math.random() noise. Individual agent dots spawn at parking/transit hubs, pathfind across the same Dijkstra graph that powers Wayfinding, and walk node-by-node to their gate — watch them live on the Venue Map. Each arrival is what increments the entry counter. It's a small, honest simulation you can actually observe, not a number generator dressed up as one.

3. Explainable AI — reasoning traces, not just conclusions

Every AI Decision-Support alert shows its reasoning trace beneath the message: the signal that triggered it, how alternate gates were compared, the trend behind it, and the resulting decision — in that order, in monospace, like a visible chain of thought. Most demos show only the final sentence; this one shows the work.

4. Forecasting, not just thresholds

Every gate keeps a rolling density history. A simple linear-trend model projects whether it's on track to cross the 85% threshold and roughly when — shown on the gate detail panel and folded into the AI Shift Briefing ("projected to cross 85% in ~7 min"). That's the difference between reacting to a problem and calling it before it happens.

Honest scope

This is a small, dependency-free static build — plain HTML/CSS/JS across three files (index.html, logic.js, and a tests.html unit-test suite), no backend, no database, nothing persists across a reload. The AI Concierge and every recommendation engine run on a built-in, transparent rule-based simulation so the whole thing works fully offline; the code is structured so a real deployment could swap in a live Claude model behind the same functions with no UI changes. Match scores, teams, and fictional gate names are illustrative — the venue data, pathfinding, and simulation mechanics are not.
